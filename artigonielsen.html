Um guia completo para transformar
qualquer interface em um pesadelo.

09 JUL 2026 • Upixel

Existem centenas de artigos ensinando como aplicar as heurísticas de Nielsen. Então resolvi fazer diferente. Se você quer construir uma interface inesquecível pelos motivos errados, este guia é para você.

1. Nunca mostre o que está acontecendo.

O botão foi clicado? Não diga nada. Deixe o usuário se perguntando se clicou ou se o sistema resolveu tirar um café.

2. Use termos que ninguém entende.

Troque 'Salvar' por 'Persistir Entidade'. Se possível, use três siglas em sequência."

3. Não permita voltar.

Errou? Problema do usuário. Toda decisão deve ser definitiva."

4. Faça cada tela diferente.

O botão Confirmar pode ser verde numa tela, azul na outra e vermelho na terceira. Surpresa mantém o usuário atento.

5. Espere o erro acontecer.

Validação? Nunca. Deixe o usuário preencher 38 campos e só avise no final que o CPF estava errado.

6. Faça o usuário decorar tudo.

Quem precisa de histórico ou sugestões? Se ele esqueceu, que comece de novo.

7. Esconda as funções importantes.

O botão principal deve ficar escondido atrás de um menu dentro de outro menu.

8. Coloque informação em todo lugar.

Se sobrou espaço na tela, coloque um gráfico, dois cards e um GIF.

9. Mensagens de erro misteriosas.

Erro 0x800A4F7B. Boa sorte.

10. Ignore completamente a ajuda.

Se precisar explicar como usar, esconda a documentação em um PDF de 240 páginas.

Se você leu este artigo e pensou 'caramba. Eu já usei um sistema exatamente assim', parabéns. Você acabou de encontrar alguém que seguiu esse manual à risca.
